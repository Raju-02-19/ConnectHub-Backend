# Project File Structure

This document describes the main files and folders for this workspace (backend Spring Boot application).

Root
- `.gitignore`: files and directories excluded from Git
- `.gitattributes`: Git attributes for line endings and file handling
- `.mvn/`: Maven wrapper support files
- `mvnw`, `mvnw.cmd`: Maven wrapper scripts
- `pom.xml`: Maven project descriptor
- `HELP.md`: Project help notes
- `PROJECT_STRUCTURE.md`: Alternative project structure documentation

src/
- `main/`
  - `java/com/chatapp/backend/`
    - `BackendApplication.java`: Spring Boot application entry point
    - `controller/`: REST controllers
      - `AuthController.java`: authentication endpoints
      - `UserController.java`: user CRUD and profile endpoints
      - `FriendController.java`: friend management endpoints
      - `MessageController.java`: message APIs
    - `dto/`: Data transfer objects for requests and responses
      - `LoginRequest.java`, `RegisterRequest.java`, `UpdateProfileRequest.java`
    - `model/`: JPA entities and domain models
      - `User.java`, `Friend.java`, `FriendRequest.java`, `Message.java`
    - `repository/`: Spring Data repositories
      - `UserRepository.java`, `FriendRepository.java`, `FriendRequestRepository.java`, `MessageRepository.java`
    - `security/`: JWT and security configuration
      - `JwtFilter.java`, `JwtUtil.java`, `SecurityConfig.java`
    - `service/`: Business logic services
      - `AuthService.java`, `UserService.java`
    - `websocket/`: WebSocket controllers and message models
      - `WebSocketConfig.java`, `ChatController.java`, `ChatMessage.java`
  - `resources/`
    - `application.properties`: runtime configuration
    - `static/`: static assets
    - `templates/`: template views (optional)

- `test/`
  - `java/com/chatapp/backend/`: test sources
    - `BackendApplicationTests.java`

target/ - build outputs generated by Maven

Notes
- Use `src/main/java/...` for production source code and `src/test/java/...` for tests.
- `application.properties` contains runtime settings such as server port and datasource configuration.
- WebSocket code is implemented in the `websocket` package alongside REST controller packages.
